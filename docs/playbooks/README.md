## Contract

Each playbook follows a fixed structure and the `ci / lint-instructions`
job enforces it:

- `## Triggers` — when the playbook applies.
- `## Steps` — what to do, in order.
- `## Definition of Done` — checklist whose CI bullets reference real
  workflow files in `.github/workflows/` (`<workflow> / <job>`).
- `## References` — at least three primary citations with retrieval
  dates.

Adding or renaming a playbook requires editing both the router and the
agent-instruction surfaces (`AGENTS.md`,
`.github/copilot-instructions.md`); the lint job enforces the link.
