# 04 — Prompt or schema change

> **Sensitive surface.** Any edit under `prompts/`, `prompts/system/`,
> `schemas/`, `docs/models.allowlist.yaml`, or `docs/mcp.allowlist.yaml`
> changes the typed contracts the LLM stages depend on. **Never
> agent-eligible.** Requires the multi-model judge artifact, a paired
> ADR, and a golden-run delta.

## Triggers

- Diff includes any file under `prompts/`, `schemas/`, or
  `docs/{models,mcp}.allowlist.yaml`.
- Diff includes the agent-instruction surfaces `AGENTS.md`,
  `.github/copilot-instructions.md`, or
  `prompts/system/agent-preamble.v1.md`.

## Pre-flight: this is human-only

If you are a cloud agent and the router landed you here, **stop**.
Comment on the issue with the sensitive-path one-liner and unassign.
The squad-bootstrap eligibility rules in `docs/squad.md` exist
specifically to prevent this case; if an issue you were assigned wants
this playbook, the eligibility gate failed and the failure must be
fixed in `ROADMAP.md` instead.

## Steps (human-author only)

1. **Run `10-research-and-decide.md` first** for any non-trivial
   structural change. Produce the per-model markdown set, the
   `decision.json` aggregating verdicts, and an ADR under `docs/adr/`.
   Skip only for narrow, additive edits (e.g. adding an enum value with
   no consumer change).
2. **Edit the source.** Change *one* schema or *one* prompt at a time.
   Cross-cutting renames belong in their own PR.
3. **Update every consumer.** Schema changes ripple through:
   - the orchestrator stage that produces the artefact;
   - the orchestrator stage that consumes it;
   - the golden run(s) under `evals/golden/`;
   - the autogenerated `docs/generated/schemas.md` (regenerate via
     `python scripts/gen_docs.py`).
4. **Multi-model judge.** Produce `judge-verdicts.json` from at least
   two distinct providers per `docs/multi-model-judge.md`. Default
   threshold is unanimous PASS per criterion; below that, a human
   resolves.
5. **Golden-run delta.** Re-run `python evals/replay.py` and confirm
   the diff is intentional. Commit the updated checkpoint blobs.
6. **Frontier-model attestation.** Tick the box in the PR template that
   declares no LLM call was added to a destructive path.

## Definition of Done

- [ ] `judge-verdicts.json` attached to the PR with verdicts from ≥ 2
      distinct providers, all rubric criteria PASS at the configured
      threshold.
- [ ] A paired ADR under `docs/adr/` is in the same PR (or for narrow
      additive changes, the rationale is captured in the rubberduck).
- [ ] `ci / orchestrator (lint + test)` passes.
- [ ] `ci / schemas (validate examples)` passes.
- [ ] `ci / lint-instructions` passes.
- [ ] `docs / generate-and-check` passes.
- [ ] `eval / offline` passes (golden-run delta intentional and
      committed).
- [ ] `rubberduck / check` passes.
- [ ] CODEOWNERS reviewer (NetSec for allowlist files; platform for
      schemas / prompts) approved.
